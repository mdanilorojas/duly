Todas las decisiones de diseño de 19 unidades, para aprobar una vez y soltar el loop

Tú elegiste: diseño batch → 1 aprobación → autónomo. Este doc fija framework, API, estados y test de cada componente del ladder §07. Al aprobar, el loop construye unidad por unidad —test→build→verify→review→commit→log— sin más interrupciones. Skip+log si algo no pasa su gate; resumen al final.

Orden: rung 1 → 6 Reuse: TanStack · React Flow · Recharts Gate: build + test + axe verde Commit + Storybook por unidad
Cómo corre el loop

Un ciclo cerrado por unidad — nada se construye sobre algo a medias

Cada componente es aditivo (archivo nuevo). Los refactors (rung 6) mantienen la API pública estable; los tests existentes son el guard. El DataTable (rung 1) debe estar mergeado antes de que rung 6 lo reuse.

Rung 5

Núcleo industrial OT — área F

pilar nuevo · disciplina ISA-101

Regla ISA-101 (toda el área): base grayscale = superficies dark del tema, color saturado SOLO en estado anormal/accionable. Severidad nunca solo por color (color + forma + etiqueta). Encaja nativo con "gastar el accent con moderación".

AlarmChip / PriorityBadgeprimitiva base
Indicador prioridad+estado colorblind-safe. Se construye primero — lo reusan Banner y Table.
priority:"critical"|"high"|"medium"|"low" state:"unack"|"ack"|"rtn"|"shelved"
Estados
criticalhighmediumlow
Reuse: Tone map · forma por estado
cada priority×state; distinguible sin color (forma)
AlarmBannercomposición
Strip persistente con la alarma no-reconocida de mayor prioridad + conteo + ack (ISA-18.2).

Reuse: AlarmChip, Button · 0 alarmas = calmo, sin color
muestra mayor prioridad; onAck; 0→estado calmo

Lista de alarmas activas ordenable por prioridad y tiempo; virtualiza en flood.

Reuse: DataTable, AlarmChip

Una aprobación y suelto el loop

Si estas 19 decisiones te parecen bien, respóndeme "aprobado, arranca" y el loop empieza en rung 1 · DataTable y baja hasta rung 6, unidad por unidad, con commit + Storybook + gate verde en cada una, sin volver a interrumpirte.

Si quieres ajustar algo primero (una API, un framework, el orden, quitar una unidad), dímelo y lo corrijo antes de arrancar.
