Qué le falta a Studio DS para ser un sistema de diseño completo de operación de agentes en la empresa

Inventario verificado contra el código, contrastado con el estado del arte 2026 (n8n, LangGraph, Microsoft AG-UI, A2A, MCP Apps, ISA-101/18.2, Clari/Gong/Salesforce). El DS es sólido en observabilidad de un agente y parcial en compliance — pero le faltan dos pilares enteros (estrategia comercial e industrial OT) y la capa de orquestación multi-agente.

Stack: shadcn/ui + Radix + Tailwind v4 + CVA React 18/19 · RSC-ready Tokens OKLCH · 3 temas Estado: propuesta — requiere tu aprobación
01 · Diagnóstico

Un cockpit de agentes casi completo por dentro, con dos habitaciones sin construir

Contando el catálogo objetivo ampliado a los seis objetivos que definiste, el sistema cubre hoy ~13% de un DS "súper completo". Lo construido tiene buena calidad (accesible, tokenizado, con Storybook). El hueco no es de calidad — es de alcance: faltan áreas enteras.

La lectura de una frase

Studio DS hoy sabe observar y auditar UN agente corriendo un proceso tipo n8n. Para el objetivo que planteaste necesita además: orquestar un enjambre (topología, control, presupuesto), operar planta industrial real (OT/SCADA, no solo n8n), y soportar la estrategia comercial (forecast, deal rooms, pricing, ROI) — tres capas hoy en cero o casi.

02 · Verificación de stack

Qué estamos usando (lo verifiqué en el código, no adivinado)

Preguntaste si usamos "chat 100 / erradics" y que si podíamos mezclar. Respuesta: sí es una mezcla, y ya está montada. Lo que oíste como eso es shadcn/ui + Radix.

📦 packages/ui/package.json — dependencias reales

Radix UI (radix-ui, @radix-ui/react-*) — primitivas headless accesibles.

shadcn/ui — patrón "copy-owned": class-variance-authority + clsx + tailwind-merge + lucide-react sobre Radix. Las 21 primitivas viven en src/components/ui/*, no como dependencia npm — son tuyas, editables.

Tailwind v4 + tokens @studio/tokens en OKLCH (tema cockpit dark-premium, test, light). React 18/19, RSC-ready, tsup build.

Mezclar más librerías (React Aria, Tremor para charts, React Flow para grafos) es viable y recomendable para las áreas nuevas — el patrón shadcn no lo impide. Ver correcciones.

F

Automatización industrial real — OT / SCADA Nueva

0✅ · 23❌  ·  23

Por qué falta: el área A cubre automatización de proceso de negocio (n8n), no tecnología operacional. "Automatización de tareas industriales" es planta real: HMI, alarmas, activos, órdenes de trabajo, OEE, seguridad. Estándares: ISA-101 (HMI de alto rendimiento — grayscale de base, color solo en anormalidad), ISA-18.2/EEMUA 191 (ciclo de 10 estados de alarma), ISA-88 (batch), ISA-95 (jerarquía de activos), IEC 61511 (SIS).

Regalo para el tema dark-premium: la convención ISA-101 es que ~90% de la pantalla es gris neutro y el color saturado se reserva solo para estados anormales/accionables — exactamente la disciplina de "gastar el accent con moderación" del DS. El estado normal de planta ES tu paleta de superficies oscuras.

06 · Qué correcciones necesita lo que existe

Correcciones — no todo faltante es nuevo; algo construido ya arrastra deuda

Dos clases: bugs verificados en el código y drift de estándar (nombres/patrones que 2026 ya estandarizó y el DS aún no adopta).

Bug · verificado

Ruta absoluta de Windows hardcodeada en el auditor a11y

scripts/a11y-audit.mjs:7 → const ROOT = 'C:/dev/Enterprise Design System'. No portable a CI ni Linux; el "76 stories, 0 violaciones" no es re-verificable automáticamente. Corrección: ruta relativa al repo (process.cwd() / import.meta.url).

Bug · verificado

Ninguna tabla está virtualizada

audit-log-table.tsx:105 comenta "no virtualizada todavía"; igual ExecutionHistoryTable. Renderizan todas las filas — no escala a volúmenes enterprise (miles de runs/eventos). Corrección: extraer un DataTable denso virtualizado (área D) y montar ambas encima. Desbloquea A, C, E, F.

Drift · nombre

"AgentCard" colisiona con el estándar A2A

En 2026 "Agent Card" = documento A2A de discovery (skills/endpoint/auth en /.well-known), no una tarjeta visual. Corrección: renombrar la presentación a AgentTile y añadir un A2AAgentCardViewer real (área G).

Drift · estándar

RichToolCallCard debería renderizar vía MCP Apps

MCP-UI + OpenAI Apps SDK se fusionaron en MCP Apps (ratificado 2026-01-26; soportado por ChatGPT, Claude, VS Code). Corrección: re-fundar el render sobre iframe sandbox + postMessage de MCP Apps, no un renderer propio.

Drift · estándar

StreamingMessage debe conformar AG-UI

El transporte agente→frontend estándar es AG-UI (~16 eventos tipados; CopilotKit, MS Agent Framework, LangGraph, CrewAI). Corrección: emitir/consumir eventos AG-UI en vez de streaming bespoke.

Drift · estándar

TraceTree: adoptar OTel GenAI e ir cross-agent

El estándar 2026 es OpenTelemetry GenAI (gen_ai.*: spans de agent/team/task/artifact/memory). Corrección: hacer los spans OTel-conformes y soportar traces distribuidos multi-agente (ver CrossAgentTraceWaterfall).

Drift · modelo

AgentHandoffMarker y CheckpointBadge están sub-modelados

AgentHandoffMarker: un marcador no captura el ciclo A2A (submitted/working/input-required/completed/canceled/failed) — parear con A2ATaskFlowView. CheckpointBadge: distinguir "checkpoint" (snapshot LangGraph) de "durable execution" (Temporal) para que ops no asuma reanudación crash-safe.

Drift · FinOps

TokenCostMeter es solo per-run

Correcto pero incompleto: 2026 exige enforcement de presupuesto cruzado — un prompt puede fan-out exponencial y sobregirar antes de que el meter per-run actualice. Corrección: alimentarlo a BudgetCapGovernor (área G).

07 · Recomendación (no ejecutar aún)

Si aprobaras, este sería el orden — máximo desbloqueo por build

Ordenado por ROI: cuánto reutiliza lo existente × cuántas áreas desbloquea × credibilidad enterprise. Es recomendación, no ejecución.

  1. DataTable denso virtualizado (área D · corrige 2 bugs)

    Primitiva única que resuelve la falta de virtualización y la reutilizan A, C, E, F. Mayor efecto palanca de todo el catálogo.

  2. Capa de orquestación mínima (área G)

    AgentTopologyGraph + SwarmControlBar + BudgetCapGovernor — cierra el hueco más grande de tu objetivo "orquestación", sobre React Flow. Sin esto el DS solo ve un agente.

  3. Cerrar compliance (área C)

    EvidenceExportDialog + ApprovalChainStepper + ModelProvenanceCard reutilizan actor-dual/hash de AuditLogTable. Sostiene la credibilidad ante auditor (EU AI Act, SOC2).

  4. Núcleo comercial (área E — pilar nuevo)

    ForecastRollupTable · PipelineWaterfallChart · MutualActionPlanBoard · RelationshipMap · PricingApprovalMatrix · MRRMovementWaterfall: los 6 que definen la dimensión comercial, sin equivalente en dashboard genérico.

  5. Núcleo industrial OT (área F — pilar nuevo)

    AlarmBanner · AlarmSummaryTable · ProcessValueTile · AssetHealthGauge · OEEWaterfall · AutonomyModeSwitch con disciplina ISA-101 grayscale — encaja natural en el tema dark.

  6. Adopción de estándares en lo existente (correcciones)

    MCP Apps en RichToolCallCard, AG-UI en streaming, OTel GenAI en TraceTree, renombrar AgentCard→AgentTile. Deuda que crece si se pospone.
