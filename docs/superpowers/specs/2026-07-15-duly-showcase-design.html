Decisiones clave

Ubicación

Dentro del monorepo (apps/showcase), no repo separado. @duly/ui no está publicado a npm — un repo independiente no podría instalarlo. Consume workspace:* igual que Storybook.

Stack

Vite + React + TS, mismo patrón zero-Tailwind que examples/minimal-vite: importa @duly/ui/styles.css directo, tema vía data-theme. Sin dependencias nuevas.

Datos

Escritos frescos dentro de apps/showcase, uno por página — vocabulario modelado sobre las stories existentes, pero no importado (los consts no están exportados; exportarlos tocaría ~30 archivos fuera de alcance). Duplicación mínima aceptada por mantener la app autocontenida.

Navegación

Estado local, sin router — 5 vistas no lo justifican. Extender = agregar página + link en el nav, sin tocar el resto.

Deploy

apps/showcase/ (root directory del 2do proyecto Vercel) └─ vite build → dist/ Proyecto Vercel "duly-sc" (mismo team, mismo repo GitHub "duly") └─ dominio duly-sc.auraby.design (agregado vía Vercel CLI, no /adeploy)

/adeploy no aplica — asume carpeta → repo GitHub nuevo independiente. Acá es un 2do proyecto Vercel sobre el repo existente, mismo patrón que ya usa "duly" (proyecto Storybook) con su propio root directory.
